# 文档风格变更

## 目标

将 freerec 文档站点的视觉风格与个人博客 (mtandhj.github.io) 对齐，保持统一的品牌感。

## 参考博客风格

| 项目 | 博客现状 |
|------|----------|
| 技术栈 | Hugo 自定义主题 + Tailwind CSS |
| 配色 | 大地色系：石灰底 `#f0eeec`，棕色强调 `#4b3425`，暗色 `stone-800/900` |
| 字体 | Rubik (Google Fonts) |
| 代码高亮 | Monokai |
| 风格 | 极简，无多余装饰 |

## 变更内容

### 1. 配色对齐

- **主色调**：像素橙 `#e07020` → 棕色 `#4b3425`
- **背景色**：白色 → 石灰 `#f0eeec`（亮色）/ `stone-800` `#292524`（暗色）
- **链接色**：棕色 `#4b3425`，hover 变灰 `#c0c0c0`
- **代码高亮**：保持 Monokai（暗色模式已使用 Monokai，亮色模式也统一）

### 2. 字体对齐

- 全局字体换为 **Rubik**（通过 Google Fonts CDN 引入）

### 3. 页面布局对齐

- 去掉侧边栏的固定宽度限制，与博客保持类似的内容/留白比例
- 恢复 PyData 主题默认的侧边栏宽度（去掉之前的自定义宽度覆盖）

### 4. 去除像素风元素

- 删除所有像素风 CSS（像素边框、方角按钮、点状分隔线等）
- Logo 改为 Rubik 字体纯文字 "FreeRec"（不用图片）
- 删除像素风 logo 图片

### 5. 中英双语支持

- 导航栏添加 **中/EN 语言切换按钮**
- 两套独立页面：`/zh/` 和 `/en/`
- 中文版：教程、安装指南、快速上手、示例（当前已有内容）
- 英文版：对应翻译，API Reference 共享（来自 docstring，本身是英文）
- 通过 Sphinx 的 i18n（gettext）机制实现，或双目录构建

### 6. 不变的部分

- Sphinx + PyData 主题框架不变
- 文档结构、API Reference 不变
- GitHub Actions 部署不变
- 暗色模式支持保留
